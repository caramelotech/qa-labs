## Exercício 01 - Identificando Riscos

**Objetivo:** Praticar o levantamento de riscos sob a perspectiva do usuário final.

**Instrução:**
Escolha um aplicativo que você usa no dia a dia (ex: app de banco, e-commerce, rede social). Liste ao menos **5 riscos** que você identificaria como tester antes de um deploy.

Para cada risco, descreva:
1. O que pode dar errado
2. Qual o impacto para o usuário
3. Como você testaria para mitigar esse risco

**Critérios de sucesso:**

- [ ] Pelo menos 5 riscos identificados
- [ ] Cada risco tem impacto e estratégia de teste descritos
- [ ] Os riscos consideram diferentes camadas (UI, backend, dados)

## Exercício 02 - Modelo Ágil vs. Tradicional

**Objetivo:** Compreender as diferenças de atuação do QA em cada modelo.

**Instrução:**
Com base no que você aprendeu, compare a atuação de um QA no modelo Tradicional e no modelo Ágil respondendo às seguintes perguntas:

1. Em qual modelo o QA participa mais cedo do processo?
2. Em qual modelo a responsabilidade de qualidade é mais compartilhada?
3. Quais artefatos são comuns aos dois modelos?

**Critérios de sucesso:**

- [ ] Todas as perguntas foram respondidas
- [ ] As respostas demonstram compreensão das diferenças entre os modelos
